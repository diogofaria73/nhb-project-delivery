# US-07 — Structured Excel Exports

## Context

The platform currently exports analytics (US-05.5) and goal breakdowns (US-06.3) as **CSV**. Although CSV is portable, it lacks structure: there is no header styling, no sheet separation, no conditional formatting for status bands and no totals row. Stakeholders consuming these files (Operations, Management) report that the output is **"too raw"** and requires manual cleanup before being usable in business reviews.

This US replaces every CSV export in the system with **structured XLSX (Excel) workbooks**. Each workbook follows a consistent layout — cover sheet with metadata, structured data sheets, header styling, frozen panes, conditional formatting, totals — so the file can be opened and shared as-is.

Two exports are in scope:

- **Analytics export** (replacing US-05.5 / `GET /api/status-reports/analytics/export`).
- **Goal breakdown export** (replacing US-06.3 / `GET /api/goals/:id/export`).

The CSV endpoints are **removed**; the new endpoints return `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` with the `.xlsx` extension.

## US-07.1 — Analytics export to Excel

**As an** Administrator,
**I want to** download the status-report analytics as a structured Excel workbook,
**So that** I can share the file in business reviews without manual cleanup.

### Acceptance Criteria

- [ ] The **"Export"** dropdown on `/status-reports/analytics` now offers **Excel** (flat) and **Excel** (summary) instead of CSV
- [ ] Each option produces a `.xlsx` file with the filename `analytics-{from}-{to}-{kind}{-companyId?}.xlsx`
- [ ] The workbook always opens on the **Cover sheet** (sheet 1)
- [ ] **Cover sheet** contains:
  - Title row: "Status Report Analytics" in 16 pt bold
  - Generation date in user's locale + UTC timestamp
  - Window (from–to in `MMM/YYYY` format)
  - Scope: "All companies" or the trade name when filtered
  - Active goal (if any): `Goal: {{deliveriesPerPeriod}} per period · deadline day {{monthlyDeadlineDay}}`
  - KPI table (4 rows): Expected, On-time, Late, Missed — each with absolute and % of expected
- [ ] **Flat** workbook additionally contains:
  - Sheet **"Deliveries"** with one row per `(company, referenceMonth)`
  - Columns: Company, Reference month (`MMM/YYYY`), Status, Submitted at (locale + UTC ISO), Delivery email
  - Top row frozen; auto-filter on the header
  - Status cells colored by band (on-time: green / late: amber / missed: red / inactive: gray)
  - Sheet **"Monthly summary"** with one row per month (Expected / On-time / Late / Missed + on-time %) and a totals row
- [ ] **Summary** workbook additionally contains:
  - Sheet **"Per company"** with columns: Company, Expected, On-time, Late, Missed, Score (%), Band
  - Band column colored as in the flat workbook
  - Totals row at the bottom (sum of Expected / On-time / Late / Missed; weighted Score)
  - Conditional bar in the **Score** column (data bar) for at-a-glance comparison
- [ ] All numeric cells use the **tabular-number** format (`#,##0` or `0.0%`)
- [ ] Column widths are auto-fitted at generation time (no manual resize needed)
- [ ] The workbook respects the **company scope filter** active on the screen
- [ ] When the period has **no deliveries**, the workbook still generates with the cover sheet and an empty Deliveries / Per-company sheet plus a centered note "No deliveries logged in this window."

### Business Rules

- BR-45: Analytics exports return `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` (not CSV). The previous CSV endpoint and the `kind=flat|summary` query parameter are preserved (CSV is removed).
- BR-46: Every workbook generated by the platform follows the **common style guide** (BR-49) — same colors, fonts, header treatment, and cover sheet — regardless of which use case generated it.

## US-07.2 — Goal breakdown export to Excel

**As an** authenticated User,
**I want to** download a goal's breakdown as a structured Excel workbook,
**So that** I can use the file as input for follow-ups with the companies that are off-track.

### Acceptance Criteria

- [ ] The "Export CSV" button on `/goals/:id` becomes "Export Excel" and produces `goal-{periodLabel}-breakdown.xlsx`
- [ ] **Cover sheet** contains:
  - Title row: "Goal Breakdown — {{periodLabel}}"
  - Generation date + UTC timestamp
  - Goal metadata: Period (start → end), Deliveries per period, Monthly deadline day, Notes (if any)
  - Aggregate summary (4 KPI rows): Eligible companies, Companies hitting, Total delivered, Total on-time + computed hit rate %
- [ ] Sheet **"Per company"** with columns: Company, Delivered, On-time, Late, Missing, Status, Projected (Delivered / On-time / Status) when projection was requested
- [ ] Status cell colored by band (hit: green / at-risk: amber / off-track: red)
- [ ] **Missing** column shows a downward conditional arrow when > 0 (amber/red depending on amount)
- [ ] Top row frozen; auto-filter on the header
- [ ] Totals row at the bottom (sum of Delivered / On-time / Late / Missing; counts of Hit, At-risk, Off-track)
- [ ] All numeric cells use tabular-number format; status / band cells use the platform style guide colors
- [ ] When the goal has no eligible companies, the workbook still generates with the cover sheet and an empty Per-company sheet with a centered note "No eligible companies in this period."

### Business Rules

- BR-47: The goal export honors the same `project` query parameter as the JSON breakdown (US-06.3). The projection result appears in dedicated columns.
- BR-48: The goal Excel uses the same style guide (BR-49) as the analytics workbook.

## US-07.3 — Workbook style guide

**As a** platform owner,
**I want** every Excel export to follow a common visual style,
**So that** files share a consistent identity regardless of the originating screen.

### Acceptance Criteria

- [ ] A shared style guide is defined and applied by **every** XLSX-emitting use case:
  - **Header rows** — bold, font color `#FFFFFF`, fill `#0F172A` (slate 900), border bottom thin `#CBD5E1`
  - **Body rows** — alternating fill `#FFFFFF` and `#F8FAFC` (zebra)
  - **Tabular numbers** — Calibri 11, right-aligned, format `#,##0`
  - **Percent** — format `0.0%`
  - **Status cells** — fill + dark text: on-time/hit → `#D1FAE5 / #065F46`; late/at-risk → `#FEF3C7 / #92400E`; missed/off-track → `#FEE2E2 / #991B1B`; inactive → `#F1F5F9 / #475569`
  - **Cover sheet** — title 16 pt bold; metadata rows with 11 pt; KPIs in a 2×N table with light borders
  - **Frozen panes** — first row of each data sheet
  - **Auto-filter** — enabled on the header row of each data sheet
  - **Column widths** — generated server-side based on content sample
- [ ] Translations of column headers follow the current i18n language passed by the client (header `Accept-Language` or a `lang` query param)

### Business Rules

- BR-49: A central `WorkbookStyleGuide` module is the single source of truth for fonts, colors and number formats. Use cases never inline styling.

## Technical Notes

- **Library:** `exceljs` (mature streaming XLSX library; supports styles, data validation, conditional formats and number formats). Add as a dependency of `apps/api`.
- **Module location:** introduce `apps/api/src/shared/infrastructure/exports/` with:
  - `workbook-style-guide.ts` — colors, fonts, number formats (BR-49)
  - `excel-writer.helpers.ts` — header row builder, totals row builder, conditional band painter, auto-width helper
- **Use cases (refactor):**
  - `AnalyticsExportUseCase` → returns a `Buffer` (or pipes a `Workbook.xlsx.write(stream)`) instead of a CSV string. Two kinds: `flat` and `summary`. Cover sheet built from `OverviewResponse`.
  - `ExportGoalBreakdownUseCase` → same shape; builds cover sheet from `goal` + `breakdown.summary`, per-company sheet from `breakdown.rows`.
- **Controllers:** `Content-Type` becomes `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`. `Content-Disposition: attachment; filename="…xlsx"`. Stream the workbook to the response instead of holding it in memory.
- **i18n on backend:** add a thin translation layer for header labels (one map keyed by language). The client sends the active language as `?lang=en` / `?lang=pt-BR`.
- **Frontend:**
  - Replace "Export CSV" labels with "Export Excel" and update the dropdown copy.
  - Filename hint is provided server-side via the `Content-Disposition` header; no client change needed beyond label.
  - Existing `apiClient.getBlob` already works for binary downloads.

## Out of Scope (for this US)

- **Embedded native Excel charts** (pie / bar) — text + conditional formatting is enough for V1. A future US can introduce charts.
- **Server-side rendering of the heatmap into a sheet** — heatmap is excluded from V1.
- **Password-protected workbooks** — out of scope.
- **PDF export** — separate US.
- **Localizing numeric formats** (decimal separator by locale) — Excel respects the user's locale settings at open time; we always emit US conventions in the file.
